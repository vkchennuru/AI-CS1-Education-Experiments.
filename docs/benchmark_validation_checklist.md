# Benchmark Validation Checklist

Before a problem is accepted into the benchmark, verify:

- [ ] Learning outcome is clearly defined.
- [ ] Bloom's level is appropriate.
- [ ] Difficulty is appropriate.
- [ ] Problem statement is unambiguous.
- [ ] Input format is complete.
- [ ] Output format is complete.
- [ ] Constraints are realistic.
- [ ] Sample input/output is correct.
- [ ] Hidden test cases cover normal cases.
- [ ] Hidden test cases cover boundary cases.
- [ ] Hidden test cases cover edge cases.
- [ ] Reference solution passes all hidden tests.
