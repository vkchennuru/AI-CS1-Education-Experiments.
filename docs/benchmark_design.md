## Purpose

This benchmark has been developed to systematically evaluate AI-generated Python solutions for introductory programming (CS1) problems.

The benchmark is designed to investigate whether novice programmers can reliably trust AI-generated code and to motivate the inclusion of code judgment skills in introductory programming education.

---

# 1. Benchmark Construction Criteria

The benchmark was designed using a systematic educational and methodological framework rather than by selecting an arbitrary number of programming problems.

The following criteria guided the construction of the benchmark.

| Criterion | Description |
|-----------|-------------|
| Curriculum Coverage | Covers the complete introductory Python programming (CS1) syllabus. |
| Bloom's Taxonomy | Includes programming problems across all six cognitive levels. |
| Difficulty Levels | Includes Easy, Medium, and Hard programming problems. |
| Programming Constructs | Covers fundamental programming concepts including variables, control structures, functions, collections, recursion, file handling, and object-oriented programming. |
| Problem Diversity | Includes numerical computing, text processing, data structures, algorithmic thinking, and real-world programming tasks. |
| Learning Outcomes | Every problem is aligned with one or more CS1 learning outcomes. |
| Hidden Test Cases | Every benchmark problem contains hidden test cases covering normal, boundary, and edge-case scenarios. |
| Reproducibility | Every problem is evaluated using the same prompt template, execution environment, and automated evaluation framework. |
